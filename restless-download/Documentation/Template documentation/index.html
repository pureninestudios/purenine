HTML Structure

In this section we will present you the main HTML structure of all files.

Restless HTML template comes with many layout options but there are two different HTML structures, first with standard top header and footer, and second with left navigation menu without footer. We will explain all parts of HTML structure later, now we will show you the basic setting of it.

So what you need to know? As any HTML file we have head and body tag. Inside of the head tag we have some meta information about the template, like title, description, author and tags. This is important for SEO purposes. Every page of your website should have unique title and description. Also, inside of the head tag we have calls for stylesheets, template css files and Google web fonts.

Ok now we know what is in the head tag. Let's see what is in body tag. There is your content and at the bottom there is some script tags, used for calling specific jQuery plugins and scripts. Those are needed for specific elements on the page to function, which we will talk about a bit later.
So how is the content organized. Inside of the body we first have header. Header contains logo, main navigation and search. As this is HTML template search is not functional by default. But you can use custom Google search if you need that functionality.
Next thing we have in the body tag is content, actual content of your site. Every peace of content is placed inside a div with class "page-content". Page content can have background color or background image. You can read more about page-content settings in page content settings section . Inside of page content we have classical Bootstrap structure. That is container, div and the columns. Inside the columns you can place content elements. You can have more than one page content on a page. Consider it a section of content. At the end we have footer containing some widgets and copyright info. So this is basic HTML structure of the template.

Every layout has specific color style. Each color style is placed into separate css file. You can find it in css folder. Color files are named "color-*.css". You can edit css file that is used in the layout, so you don't need to replace the file in each HTML file. So for example home-blog page uses color-blue css file. You can simply open that css file and replace the green values with the color of your brand.

Understanding the grid

As we already stated, Restless is based on Bootstrap 3. This means that it uses Bootstrap's 1170px grid with 12 columns. Each column is placed inside the row. Row is placed inside a container, which can be fixed width - 1170px or fluid - takes whole page width. To use fluid container simply use class "container-fluid". Container is placed inside page-content. That is the main structure you must follow.

Elements comes inside a column. You can combine columns in a way that sum of their numbers is 12. So for example you can have two col-md-6 in one row. or you can have three col-md-4 in one row.

You can have more than one row inside a container. Also, you can have more than one page content inside a page. Page content is just a section of content on which you can add background image or color. So if you have a part of a page that has some custom background, place it in a page-content and give that page content a class that defines it's background. More about page content element in the next section.

You can read more about Bootstrap grid and how you can combine and use columns on their official pages.

Page content backgrounds

As you can see in the demo files, some sections - page-content divs have image or color background. This is defined in style.css file so you can make customizations there.

So how do you know that page-content has a custom background? We have custom classes for that. First of all we will have class "custom-background". That class defines custom padding on that section. After that we can have a class that defines a background image or color. For example class "bkg-img2" defines that page-content will have image background which is defined by that class in style.css file. Also, when we have a page-content that has some type of custom background we need to say that it needs to have bottom margin, so that the content that comes after doesn't stick to it. We have couple of helpers classes for that, but we always used (in this template), class "mb-120", which says margin-bottom is 120px.

You can read more about custom, helpers classes in helper classes sections.

Navigation customization

We have three different navigation structures depending on header type. The first one is for classic menu, second one for hamburger menu and the third for left menu. Here we will explain how to customize it, that is how to create your own navigation.

Classic header navigation

So the navigation is placed inside ul with class "navbar" and "nav". Every list item inside is first level menu item. The class "current-menu-item" is used for styling current page. So this is example for home page. For about us page, you will place this class on list item where link for about us page is.

This navigation has data hover effect so you need to enter the value of the link in data attribute called data-hover.

If you need second level navigation, a dropdown, then you need to use class "dropdown" and class "menu-item-has-children" which will add that little arrow on the menu item. if you need third level navigation you need to use class "dropdown" and "dropdown-submenu" class. You add those classes on list item (menu item) that will have submenu.

Hamburger header navigation

In hamburger header navigation is placed inside of element with "fat-nav" class. Every list item inside is first level menu item.

For second level navigation you should use class "dropdown" on a link and add list with class "hamburger-nav-submenu" which will contain drodpown menu items. This structure is defined by fatNav plugin.

Left menu navigation

In left menu header navigation is placed inside element with id "ml-menu". This navigation is multi level menu plugin so it has defined structure. Each menu item has class "menu__item" and contains link with class "menu__link". If you want dropdown menu link should have data attribute with submenu name for example data-submenu="submenu-1". Submenu is unordered list with data attribute data-menu="submenu-1" (notice the connection between them).

How to make your site retina ready

This is pretty simple. On Restless we used svg format for every icon, and font icons, so every icon will look sharp on each screen. So you don't have to do anything here.

What you need to do is to make your photos retina ready. So each photo you use, make one that is twice bigger. Here you have some naming standard that you will have to use, because of the retina.js script that will handle the switch of images on retina screens. Yes, script will do all the job needed in the back. So what you need to do is following. let's say you have image that is called image01.jpg. This image is 400x600px. make the same image in double size, that is 800x1200 and name it image01@2x. This part is important. Your double size image needs to have that @2x part. Also those images needs to be placed inside the same folder. The last thing you need to do is include retina.js and retina.css script on your page. Retina.css file contains background images so you should define them double sizes. For example if icon is 20x20px you should define background size 40x40px.
